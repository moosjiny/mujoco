# 듀얼 암 인터랙티브 제어(IK) 트러블슈팅 및 해결 과정 가이드

본 문서는 MuJoCo 환경에서 듀얼 암(Dual Arm)의 역기구학(Inverse Kinematics, IK) 기반 제어와 ROS2 Teleop 연동 과정에서 발생했던 주요 문제들과 그 해결책을 기록한 문서입니다. 향후 유사한 문제 발생 시 참고 자료로 활용하기 위해 작성되었습니다.

---

## 1. 초기 상태: Mocap 구슬 드래그 불가 현상
**[문제 상황]**
사용자가 `Ctrl + 우클릭`으로 타겟 구슬(Mocap body)을 드래그하려 했으나 전혀 움직이지 않는 현상 발생.

**[원인 1: 크기와 겹침 문제]**
- Mocap 구슬의 크기(반경 4cm)가 로봇 손목의 마지막 관절(`left_link7`) 크기와 비슷하거나 작았으며, 위치가 완전히 동일하게 동기화되어 있었습니다.
- 사용자가 구슬을 더블클릭할 때, MuJoCo 뷰어의 마우스 레이캐스팅이 구슬이 아닌 바깥쪽을 감싸고 있는 **회색 로봇 팔(Rigid Body)**을 선택해버렸습니다.

**[해결 1]**
- `build_mjcf.py`에서 Mocap 구슬의 크기를 `size=[0.08, 0, 0]` (반경 8cm)로 2배 이상 키워 시각적으로 손 밖으로 돌출되게 하여 쉽게 클릭할 수 있도록 수정했습니다.

**[원인 2: Perturbation 누락 버그 (핵심)]**
- 구슬을 크게 키우고 ROS2 Teleop을 연동한 뒤에도 여전히 마우스 드래그로는 타겟이 이동하지 않았습니다.
- Python으로 작성된 커스텀 `mj_kinematics` 반복 루프 안에서 사용자의 **마우스 드래그 데이터(Perturbation)를 Mocap 좌표계에 반영하는 함수 호출이 누락**되어 있었습니다.
- 즉, 화면에는 클릭된 붉은 상자가 보였으나, 물리 엔진 상의 실제 타겟 좌표(`data.mocap_pos`)는 업데이트되지 않았습니다.

**[해결 2]**
- `sim_ros2_ik.py` 루프 내에 `mujoco.mjv_applyPerturbPose(model, data, viewer.perturb, 0)` 함수를 추가하여 마우스 드래그 위치를 즉각적으로 물리 엔진의 Mocap 위치에 반영하도록 수정했습니다.

## 2. 역기구학(IK) 이동 불가 및 과도한 진동(Shaking) 현상
**[문제 상황]**
Mocap 타겟과 로봇 손의 위치가 벌어질 때 로봇 팔이 덜덜 떨리거나 엉뚱한 방향으로 발산하며 멈추는 현상 발생.

**[원인 1: 6D IK의 쿼터니언(회전) 강제 매칭 문제]**
- 초기에는 위치(Position)와 회전(Rotation)을 모두 맞추는 6D IK를 적용했습니다.
- 타겟 구슬의 초기 회전값이 `[1, 0, 0, 0]`으로 설정된 반면 실제 로봇 손목은 꺾여있었기 때문에, 시작 직후 엄청난 회전 오차를 보정하려다 관절 한계(Joint Limit)에 부딪혀 시스템이 락(Lock)에 걸렸습니다.

**[해결 1]**
- 보다 안정적인 3D **Position-only IK** (위치 추적 전용 역기구학)로 다운그레이드하여 위치만 따라가도록 단순화 및 안정화했습니다.

**[원인 2: 잘못된 속도 제한(Velocity Clipping)]**
- 팔이 너무 빨리 움직일 때 진동하는 것을 막고자 `dq_side = np.clip(dq_side, -0.2, 0.2)` 형태로 개별 관절의 속도를 잘라냈습니다.
- 이 방식은 조인트 속도 벡터의 방향 자체를 왜곡하여, 팔이 직진하지 못하고 엉뚱한 방향으로 도망가게 만들었습니다.

**[해결 2]**
- 벡터의 방향을 유지하면서 전체 크기만 줄이는 **Vector Scaling** 방식을 채택했습니다.
- `if dq_norm > max_dq: dq_side *= (max_dq / dq_norm)` 코드를 적용해 떨림 없이 부드럽고 신속하게 이동하도록 수정했습니다.

## 3. 기구적 간섭(Collision) 통과 현상
**[문제 상황]**
로봇 팔이 이동 시 바닥이나 반대편 팔과 겹치거나 통과해버리는 현상(Self-Collision 무시).

**[해결 과정]**
1. `build_mjcf.py`에서 꺼두었던 물리 충돌 감지 비활성 플래그(`mjDSBL_CONTACT`)를 제거하여 엔진 단에서 충돌 쌍(Contact pair)을 계산하도록 복구했습니다.
2. `sim_ros2_ik.py`에 단축키(`C` 키) 기반의 **Collision Avoidance Toggle** 로직을 도입했습니다.
3. 매 IK 연산 후 `mujoco.mj_collision()`을 호출하여 `data.ncon > 0` (간섭 발생)일 경우, 관절을 업데이트 전의 백업 위치(`qpos_backup`)로 롤백시켜 서로 파고들지 못하도록 구현했습니다.

## 4. 최종 결과 및 기능 요약
현재 구축된 스크립트(`sim_ros2_ik.py`)는 다음과 같은 기능을 동시에 완벽히 지원합니다:

- **완벽한 마우스 텔레오퍼레이션**: 뷰어 내 구슬을 더블클릭하고 `Ctrl + 우클릭 드래그` 시, 뷰어의 이동량이 즉각적으로 역기구학 솔버에 입력되어 로봇 팔이 정교하게 추종합니다.
- **ROS2 Topic 기반 제어**: 외부 노드(`teleop_twist_keyboard`)에서 퍼블리시하는 `/target_left/cmd_vel` 등의 메시지를 수신받아 마우스와 동일하게 구슬을 이동시킬 수 있습니다.
- **안전 제어**: `C` 키 토글을 통해 실시간으로 물리적 간섭을 감지하고 회피할 수 있으며, 현재 활성화 여부가 뷰어 좌측 상단에 표시(Overlay)됩니다.
