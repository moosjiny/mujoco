CAN 인터페이스(CAN 2.0 / CAN FD) 구성 후 정상적으로 설정되었는지 확인
Code
1

ip link show

초기 상태: can0 인터페이스가 DOWN 상태로 비활성화
로봇-PC 간 재연결시에 인터페이스가 비활성화로 초기화되므로 다시 인터페이스 설정을 해줘야 한다.

CAN 2.0 설정 후: 1 Mbps로 설정되며 can0 인터페이스가 UP 상태로 활성화

CAN FD 설정 후: 1 Mbps / 5 Mbps로 설정되며 can0 인터페이스가 UP 상태로 유지되고 MTU가 72로 변경

인터페이스 설정이 잘 되었는지 CAN 통신 테스트

CAN 인터페이스를 통해 모터와의 송수신 통신이 정상적으로 이루어지는지 확인
터미널 1 - CAN 0 트래픽 모니터링

Code
1

candump can0

터미널 2 - 명령 전송: 터미널 2에서 can0(Leader-R) 모터 #1에 모터 활성화 명령을 전송
정상적으로 동작을 한다면 can0의 첫 번째 모터의 LED가 적색 → 녹색으로 켜진다.
Code
1

cansend can0 001#FFFFFFFFFFFFFFFC

터미널 2 - 테스트 후 모터를 비활성화
정상적으로 동작을 한다면 can0의 첫 번째 모터의 LED가 녹색 → 적색 으로 꺼진다.

Code
1

cansend can0 001#FFFFFFFFFFFFFFFD

명령어 분석:

: Leader - R
001 : 모터 #1 CAN ID (0x01)
FFFFFFFFFFFFFFFC : 모터 활성화 명령
FFFFFFFFFFFFFFFD : 모터 비활성화 명령
can0

Baudrate 설정
모터 Baudrate 설정 시에는 CAN 2.0 (1 Mbps) 환경으로 바꿔서 진행해야 한다.
설정 완료 후에는 목표 Baudrate(CAN FD)에 맞게 인터페이스를 재설정한다.
모터 파라미터는 약 10,000회 쓰기 제한이 있으므로 반복 실행을 지양해야 한다.

모터 점검 파일 실행

모터 점검 실행 시, 명령어의 send CAN ID 와 receive CAN ID 를 각 모터에 설정된 값으로 변경하여 사용
예를 들어, CAN ID 17은 16진수 0x11을 10진수로 변환한 값

점검 확인 및 로그 해석

1. 전송 속도 확인 : 표시된 전송 속도가 이전에 설정한 속도와 일치하는지 확인
1. 전송 속도가 일치하지 않으면 CAN 인터페이스를 2.0으로 전환하고 전송 속도 변경 스크립트를 다시 실행
2. 영구 설정 테스트 : --flash 를 사용한 경우, 모터의 전원을 껐다 켜고 다시 점검을 실행하여 설정이 유지되었는지 확인
3. CAN 통신 연결 및 설정 확인
1. Control Mode가 MIT 여야 한다.
4. 모터 초기화 및 파라미터 정상 응답이 되는지 확인
5. Enabling Motor 후 상태 값(Position/Velocity 등)이 정상 수신하는지 확인

(이슈) 모터 Control Mode 오류

이슈: 모터 점검 중 Queried Control Mode (-1) is not MIT. 로그가 뜨고 다음 모터로 넘어간다.
해결법: 모터간 CAN 케이블 커넥터 연결이 분리되어 재연결하거나, 커넥터 부분 단락을 납땝해야 한다.

Zero Position Calibration
우선 팔 위치를 아래 사진처럼 대략 Zero Position으로 조정
리더-팔로워는 각각 독립적인 상태로 Calibration 작업을 시행
CAN-FD로 인터페이스를 설정한 상태에서 진행
openarm_can Python 라이브러리가 설치된 가상환경에서 진행

Follower-side Right (Defaults)
Code
1

openarm-can-zero-position-calibration --canport can0 --arm-side right_arm

Follower-side Left
Code
1

openarm-can-zero-position-calibration --canport can1 --arm-side left_arm

실행되는 모습

각 모터의 기계적으로 끝단을 찾고, 모터의 zero position을 기록합니다.
