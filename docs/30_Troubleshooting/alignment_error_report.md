# 📋 Dual-Arm System Alignment Error Report

**작성일:** 2026-04-27
**상태:** 해결 완료 (Resolved)

## 1. 문제 현상 (Issues)
- **Visual Disconnection**: Rerun 시뮬레이터 상에서 로봇 팔의 각 링크가 서로 분리되어 허공에 떠 있는 '폭발도(Exploded View)' 현상 발생.
- **Assembly Gaps**: 특히 어깨 블록(Shoulder)과 팔의 첫 번째 링크(Base Link) 사이가 수 센티미터 이상 벌어져 디지털 트윈으로서의 정확도 상실.

## 2. 원인 분석 (Root Cause Analysis)

### 2.1 STL 메쉬 원점 불일치
- 분석 결과, 제공된 STL 메쉬 파일들이 로컬 원점(0,0,0)을 기준으로 생성된 것이 아니라, CAD 조립체 상의 **절대 좌표**를 가진 상태로 익스포트됨.
- `link1.stl`의 경우, 메쉬 데이터 자체가 이미 Z축으로 **63.0mm** 지점에서 시작됨.

### 2.2 보정값 불일치 (Manual Offset Mismatch)
- 기존 `sim_dual_arm.py`에서 사용하던 `OFFSETS_LEFT/RIGHT` 사전의 값이 실제 URDF 조인트 위치와 일치하지 않음.
- URDF 상의 누적 조인트 높이는 **122.5mm**였으나, 시각화 코드에서는 **121.5mm**를 사용하여 1mm의 상시 오차 발생.

### 2.3 이중 변환 오류
- MuJoCo의 정기구학(Forward Kinematics)이 계산한 글로벌 좌표에, 메쉬 자체의 내장 오프셋이 더해지면서 팔이 원래 위치보다 훨씬 높은 곳에 렌더링됨.

## 3. 해결 방안 (Solution Details)

### 3.1 메쉬 좌표계 정밀 측정
- `trimesh`를 활용하여 각 메쉬의 실제 하단면(`Min Z`) 좌표를 전수 조사함.

### 3.2 URDF 기반 오프셋 동기화
- URDF 파일의 조인트 `origin` 수치를 누적 합산하여 물리 엔진과 시각화 엔진의 좌표계를 1:1로 매칭함.
- **최종 적용 수치:**
  - Base Link: 0.0
  - Link 1: 62.5
  - Link 2: 122.5 (62.5 + 60.0)
  - Link 3: 188.75 (122.5 + 66.25)
  - Link 4: 342.5 (188.75 + 153.75)
  - ... 이후 모든 링크 정밀 보정

### 3.3 렌더링 최적화
- `vertex_normals` 데이터를 추가하여 단색의 플랫한 화면에서 빛 반사와 굴곡이 느껴지는 프리미엄 렌더링으로 개선.

## 4. 검증 결과 (Verification)
- Rerun Viewer 접속 결과, 모든 관절의 결합 부위가 **간극 없이(Zero-Gap)** 완벽하게 연결됨을 확인.
- 팔의 움직임 시에도 메쉬가 관절에서 이탈하지 않고 정확히 추종함.
