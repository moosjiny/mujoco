### 1) OMX-F 메시 적용
**의미**: 디지털 트윈의 핵심은 실제 형상 — 박스로는 충돌/IK/워크스페이스 분석 불가.

- MJCF의 `omxf_link0~7` 녹색 박스 8개를 `follower_01_base.stl ~ follower_08_gripper_gear.stl`로 교체
- 초기엔 GitHub raw URL에서 다운로드 → `ros-jazzy-open-manipulator-description` apt 패키지 설치 후 `/opt/ros/jazzy/share/...` 시스템 경로로 전환

### 4) 수치 안정성 (NaN 해결)
**의미**: 시뮬레이션 발산 시 모든 자세/속도가 NaN → 학습/데이터 수집 모두 무력화.

- 원인: OMX-F 관절 7개에 `damping`/`frictionloss` 누락 (OpenArm은 0.25/0.02 있음)
- 모든 omxf 관절에 추가:
  ```xml
  damping="0.3" frictionloss="0.02" armature="0.001"
  ```
- 적분기 변경: `<option timestep="0.001" integrator="implicitfast"/>`
- 검증: 5초 자유 시뮬 + 5초 양팔+그리퍼+OMX-F 동시 동작 → NaN 없음

### 5) vicpinky 자유 운동 활성화
**의미**: 모바일 로봇이 월드에 박혀있으면 차동구동 의미 없음.

- `<freejoint name="pinky_base_free"/>` 추가 → nv 27 → 33
- `<inertial pos="-0.2 0 0.08" mass="15" diaginertia="0.4 0.6 0.8"/>` 추가
- **시행착오**: CoM을 `(0, -0.2, 0.15)`로 잘못 설정해서 vicpinky가 전도 → `(-0.2, 0, 0.08)` 새시 중심으로 수정, 질량 55→15kg

### 6) sim_dual_arm.py freejoint 초기화 패치
**의미**: 모델 수정만으로 부족 — 매 실행마다 `qpos[:] = 0`이 freejoint를 망가뜨림.

- `data.qpos[:] = 0.0`이 freejoint의 quaternion(w,x,y,z)을 (0,0,0,0)로 만듦 → 영벡터 quaternion은 정의 불가
- 위치도 (0,0,0)으로 reset되어 vicpinky가 양팔 베이스와 겹침
- 수정:
  ```python
  data.qpos[:] = 0.0
  pinky_jnt = mujoco.mj_name2id(model, mujoco.mjtObj.mjOBJ_JOINT, "pinky_base_free")
  if pinky_jnt >= 0:
      qadr = model.jnt_qposadr[pinky_jnt]
      data.qpos[qadr:qadr+3] = [0.0, 1.2, 0.0]
      data.qpos[qadr+3:qadr+7] = [1.0, 0.0, 0.0, 0.0]
  ```

### 7) vicpinky 궤도 컨트롤러
**의미**: 모바일 매니퓰레이션의 실제 사용 시나리오 시연 — 데이터 수집/학습용.

- 액추에이터 타입 변경: torque (`general` gainprm=10) → 속도 제어 (`velocity` kv=150)
- 마찰 튜닝:
  - 휠: `friction="2.0 0.02 0.001"` (높은 접지력)
  - 캐스터: `friction="0.05 0.001 0.0001"` (낮은 저항)
  - 휠 joint damping `0.01`
- `sim_dual_arm.py` 메인 루프에 CW 궤도 PI 컨트롤러 추가:
  - 목표: 반경 1.2m, 선속도 0.18 m/s, CW
  - quaternion → yaw 변환, 접선 방향 `atan2(-px, py)`
  - `yaw_corr = 0.6 * yaw_err - 0.4 * radius_err`
  - 차동구동 역기구학: `ω_L = (V - ω·track/2) / r`, `ω_R = (V + ω·track/2) / r`
- 검증: 50초에 384°(>1바퀴), 반경 1.23 ± 0.018m, 휠 추적률 92%
