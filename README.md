# Dual-Arm MuJoCo Digital Twin

Bimanual OpenArm + OMX-F follower arm + Vic Pinky mobile base, modelled in
MuJoCo and driven by a Rerun viewer and a FastAPI status dashboard. Optional
ROS 2 Jazzy integration with RViz interactive markers.

## Layout

```
scratch/build_mjcf.py     # Single source of truth — builds urdf/dual_openarm.xml
urdf/dual_openarm.xml     # Compiled MJCF (regenerated by build_mjcf.py)
urdf/dual_openarm.urdf    # OpenArm URDF (carries STL meshes + damping)
urdf/omx_f.urdf           # Reference only; meshes pulled from system path
urdf/vicpinky.urdf        # Reference only; chassis defined in build_mjcf.py
meshes/                   # OpenArm STL/OBJ assets
scripts/                  # Launchers and helpers
dashboard/                # FastAPI status board (port 8000)
backups/                  # Snapshots of known-good model state
```

`build_mjcf.py` loads OpenArm from its URDF, then programmatically embeds the
OMX-F kinematic chain (with system-shipped STL meshes from
`/opt/ros/jazzy/share/open_manipulator_description/meshes/omx_f/`), the Vic
Pinky chassis (top plate, aluminum pillars, casters, LiDAR mount, wheels), and
the environment (skybox, checker ground, sun light). Regenerating is a visual
no-op against the checked-in XML.

## Quickstart

Python venv (system Python 3.12):

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

OMX-F meshes require the ROS package:

```bash
sudo apt install ros-jazzy-open-manipulator-description
```

### Run

```bash
bash scripts/start_all.sh
```

Brings up:

- MuJoCo native viewer + Rerun stream → <http://localhost:9090>
- FastAPI dashboard → <http://localhost:8000>

`start_full_sim.sh` additionally launches `robot_state_publisher`, RViz 2, and
an interactive-marker IK target server. Requires ROS 2 Jazzy on `PATH` and
`pyyaml` in the venv. **Note**: the ROS 2 IK path currently crashes because
`target_left` / `target_right` mocap bodies are not in the generated model —
add them to `build_mjcf.py` before using that flow.

### Regenerate the model

```bash
python scratch/build_mjcf.py
```

Tweak OMX-F joint origins, vicpinky geometry, lighting, or damping inside
`build_mjcf.py` — it's the single source of truth.
